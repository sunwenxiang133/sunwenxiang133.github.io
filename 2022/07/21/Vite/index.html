目录结构

App.vue全局的组件,在这里写的css会影响全局

main.ts公共的ts文件,共同的api可以放到这里面去

index.html 这是和webpack不一样的地方，是通过index.html作为入口文件的，（webpack，rollup 他们的入口文件都是enrty input 是一个js文件 而Vite 的入口文件是一个html文件，他刚开始不会编译这些js文件 只有当你用到的时候 如script src=”xxxxx.js” 会发起一个请求被vite拦截这时候才会解析js文件）

package.json

vite.config.ts是vite的配置文件

npm run dev 详解

npm run dev 我们在执行这个命令的时候回去找package.json的scripts，然后执行对应的dev指令

那直接执行vite命令不是更方便吗——因为我们的电脑上没有配置过相关命令， 所以无法直接执行

在我们执行npm install 的时候会在node_modules/.bin/创建好可执行文件.bin目录吗，这个目录不是任何npm包

目录下的文件表示这是一个个软链接

所有文件的开头都是#!/usr/bin/env node这是一个脚本

当我们执行npm run xxx的时候，npm会通过软链接查找这个软链接存在于源码目录node_modules/vite

查找规则是

1.先从当前项目的node_modlue /bin去找,

2.找不到去全局的node_module/bin 去找

3.再找不到 去环境变量去找

node_modules/bin中有三个vite文件

arch当然用的也是第一个啦
