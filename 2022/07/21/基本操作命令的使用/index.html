基础了解

/ 是根目录

~ 是家目录

tab能够自动补全

向前滚动一屏，向后滚动一屏 Ctrl+F（front） Ctrl+B（behing）

clear 能把之前显示的东西都清理掉

Linux下没有消息一般是成功了，消息基本是失败的消息

不同的类型linux里面颜色不一样，文件，目录，压缩包，可执行文件等等颜色都是不同的

./ 代表在当前目录找某个指令

环境变量，为了在任何一个地方都能找到某个文件，需要配置环境变量

linux的后缀

注意文件目录的/问题

  1. linux里一切皆文件
  2. 文件前面有/代表绝对路径，没有则代表相对路径
  3. 文件的后面有/代表这个目录，如果没有代表这个文件 cd sun/ 代表的是sun这个目录，没有意指这个文件

环境变量

如果没有指令路径的话，Javac是在这几个地方去找（环境变量里面）,所以在家目录里面是找不到javac的

linux目录结构

在linux世界里，一切皆文件

层级式的树状目录结构，在此结构中的最上层是跟目录“/” 然后在在目录下再创建其他的目录

linux的目录什么东西放在什么地方是规划好的，不是自己想怎么弄就怎么弄的（Windows也是这样的，只是厂家不按照这个规则放文件）

boot 用户启动的时候用到的核心文件

root root这个用户用到的文件

dev 设备管理器，会把所有的硬件映射成一个文件进行管理

系统启动必须：

/boot：存放的启动Linux 时使用的内核文件，包括连接文件以及镜像文件。

/etc【常用】：存放所有的系统需要的配置文件和子目录列表，更改目录下的文件可能会导致系统不能启动。

/lib：存放基本代码库（比如c++库），其作用类似于Windows里的DLL文件。几乎所有的应用程序都需要用到这些共享库。

/sys：【不能动】 这是linux2.6内核的一个很大的变化。该目录下安装了2.6内核中新出现的一个文件系统 sysfs 。sysfs文件系统集成了下面3种文件系统的信息：针对进程信息的proc文件系统、针对设备的devfs文件系统以及针对伪终端的devpts文件系统。该文件系统是内核设备树的一个直观反映。当一个内核对象被创建的时候，对应的文件和目录也在内核对象子系统中

指令集合：

/bin：存放着最常用的程序和指令

/sbin：只有系统管理员能使用的程序和指令。

外部文件管理：

/dev ：Device(设备)的缩写, 存放的是Linux的外部设备。注意：在Linux中访问设备和访问文件的方式是相同的。

/media【常用】：类windows的其他设备，例如U盘、光驱等等，识别后linux会把设备放到这个目录下。

/mnt：临时挂载别的文件系统的，我们可以将光驱挂载在/mnt/上，然后进入该目录就可以查看光驱里的内容了。挂载，就是开机的时候能读其它盘的信息的那个，挂载在某个目录之下

临时文件：

/run：是一个临时文件系统，存储系统启动以来的信息。当系统重启时，这个目录下的文件应该被删掉或清除。如果你的系统上有 /var/run 目录，应该让它指向 run。

/lost+found：一般情况下为空的，系统非法关机后，这里就存放一些文件。

/tmp：这个目录是用来存放一些临时文件的。

账户：

/root【常用】：系统管理员的用户主目录。

/home【常用】：用户的主目录，以用户的账号命名的。

/usr【常用】：用户的很多应用程序和文件都放在这个目录下，类似于windows下的program files目录。

/usr/bin【常用】：系统用户使用的应用程序与指令。

/usr/sbin：超级用户使用的比较高级的管理程序和系统守护程序。

/usr/src：内核源代码默认的放置目录。

/usr/local: 另一个给主机额外安装软件所安装的目录，一般是通过编译源码方式安装的程序（安装好的放在这个目录里面）

运行过程中要用：

/var：存放经常修改的数据，比如程序运行的日志文件（/var/log 目录下）。

/proc：【不能动】管理内存空间！虚拟的目录，是系统内存的映射，我们可以直接访问这个目录来，获取系统信息。这个目录的内容不在硬盘上而是在内存里，我们也可以直接修改里面的某些文件来做修改。 不能动，动了可能就开不了机了

扩展用的：

/opt：默认是空的，我们安装额外软件可以放在这个里面。如安装oracle数据库就可放到改目录下面

/srv：【不能动】存放服务启动后需要提取的数据（不用服务器就是空）

用户管理

用户登陆和注销

基本介绍

  1. 登陆时尽量少用root账号登陆，因为它是系统管理员，最大的权限，避免操作失误。可以利用普通用户登陆，登录后再用“su-用户名”命令来切换成系统管理员身份
  2. 在提示符下输入logout即可注销用户

使用细节

  1. logout注销指令在图形运行界别无效，在运行级别3下有效（即在能看到桌面的这个情况下是无效的）

创建用户

useradd 用户名

pwd 显示当前用户所在的目录（这其实是显示绝对路径的指令）

指定/修改密码

passwd 用户名 更改用户的密码

它会提示无效的密码，但是这个密码其实已经更新成功了

删除用户

删除用户，保留家目录

userdel 用户名

删除用户及其用户目录

userdel -r 用户名

需要保留家目录嘛 一般情况下建议保留

说明

创建用户成功后，会自动创建和用户同名的家目录

可以通过useradd -d 目录 用户名 ，给创建的用户执行家目录

这个不是指定目录，然后再目录下面创建文件夹，而是直接建立一个目录作为这个用户的名称

查询用户信息

基本语法

id 用户名

切换用户

su -用户名

切换到原来的用户时用logout或exit

从低权限到高权限需输入密码，高权限到低权限无需密码

who am i

显示的是登陆到了的用户的信息

用户组

类似于角色，系统可以对有共性的多个用户进行统一的管理

操作

  • 新增组 groupadd 组名

  • 删除组 groupdel 删除组

  • 增加用户时直接加上组 useradd -g 用户组 用户名 这个用户名需要没存在 用户组要预先存在 注意是组在前，用户名在后

  • 修改用户的组 usermod -g 用户组 用户名

  • 如果新建的用户没有指定组，会新建一个同名的组

  • 一些文件“在Linux里，一切皆文件”

如何找回root密码（每个系统不一样）

本质是刚开机的时候通过单用户模式进入命令行修改密码

使用指令

不同的运行级别

0 关机

1 单用户【找回丢失密码】

2 多用户状态没有网络服务

3 多用户状态有网络服务

4 系统未使用保留给用户

5 图形界面

6 系统重启

常用运行级别是3（默认的命令行运行级别是3）和5（图形界面）

指定运行级别

运行级别的切换 init 运行级别

systemctl set-default 目标运行级别

  • multi-user.target: 运行级别3
  • graphical.target: 运行级别5

帮助指令

  • man 【命令或配置文件】 获得帮助信息
    • 按q就退出帮助界面了
    • 选项可以组合使用 ls -la ls -al
  • help 命令 获得shell内置命令的帮助信息

文件目录类

  • pwd 指令 显示当前工作目录的绝对路径

    • 绝对路径 从根目录开始寻找文件
    • 相对路径 从当前目录位置开始找文件

  • ls 【选项】【目录或者文件】

    • -a 显示当前目录所有的文件和目录，包括隐藏的
    • -l 以列表的方式显示信息
      • -lm 加上了个m代表更方面human看的方式显示

  • cd 【参数】 切换到指定目录 这里相对路径，绝对路径的切换要学会

    • cd ~ 或者 cd 切换到当前用户的家目录
    • cd .. 回到当前目录的上一级目录
      • 使用绝对路径切换到root目录，cd/root
      • 使用相对路径切换到root目录，比如在/home/tom ,cd ../../tom

  • mkdir 【选项】 要创建的目录 用于创建目录

    • -P 创建多级目录 mkdir -p /home/animal/tiger

  • rmdir 【选项】 要删除的空目录 用于删除空目录

    • 如果删除的是非空目录，用 rm -rf 要删除的目录 -f强制删除 -r联级删除 在根目录使用有奇效
    • rm -rf 【地址】

  • touch 【文件名称】 创建空文件

  • cp 【选项】 source dest 指令拷贝文件到指定目录

    • -r : 递归复制整个文件夹
    • 将 /home/hello.txt 拷贝到 /home/bbb 下 cp hello.txt /home/bbb
    • 递归复制整个文件夹 /home/bbb拷贝到 /opt目录下 cp -r /home/bbb /opt
    • 强制覆盖不提示的方法 \cp \cp -r /home/bbb /opt

  • rm 【选项】 要删除的文件或目录

    • -r: 递归删除整个文件夹
    • -f : 强制删除不提示

  • mv oldNameFile newNameFile 重命名 mv /temp/movefile /targetFolder 移动文件或移动整个目录

  • cat 【选项】要查看的文件 cat只能浏览文件，不能修改

    • -n显示行号
    • 为了浏览方便，一般会带上管道命令（即执行完一条命令之后继续执行下一条） |more

  • more 基于vi 的查看文件内容的指令

    1. 空格键 向下翻一页
    2. enter 向下饭一行
    3. q 立刻离开more，不再显示文件内容
    4. = 输出当前行的行号
    5. ：f 输出文件名和当前行的行号

  • less 与more类似，但比类强大，这个是根据显示需要加载的内容，对显示大文件有较高的效率

  • echo 【选项】 【输出内容】 输出内容到控制台

    • 用echo指令输出环境变量，eg.$PATH (记得大写
    • 输出字符串 echo “hello world”

  • head -n 5 文件 查看文件前5行

    • 如果去掉 -n 5 默认显示前10行

  • tail 和head类似，这个是显示后面5行

    • tail -f 文件 实时追踪文档的所有更新，执行这个指令系统就停止在这里一直监视tail -f 有没有更新了

  • 指令和 >> 指令 输出重定向和追加指令

    • ls -l > 文件 （将列表的内容写入文件a.txt中） 重定向是覆盖写
    • ls -al >> 文件 （将列表中的内容追加写道文件aa.txt的末尾）
    • cat 文件1 > 文件2 （将文件1的内容覆盖到文件2）
    • echo “内容” >> 文件 （追加）
    • 如果被输入的文件不存在，会自动创建

  • ln -s 【原文件路径】【软连接名】 给原文件创建一个软连接

    • 软连接就类似Windows里面的快捷方式，硬链接如果原文件删除了，硬链接仍然能够访问，软连接不行
    • 用 ls -l 进行按行输出的话，是可以看出来谁是软连接的

  • history 指令 查看系统执行过的操作

    • 后面加上数字n，看的是最后执行的n步操作

时间日期类

  • date 显示当前日期

    • date +%Y 显示当前时间
    • date +%m 显示当前月份
    • date +%d 显示当前是哪一天
    • date “+%Y-%M-%d%H:%M%S” 显示年月日时分秒 注意这个是字符串的格式 只要显示的不是单独的某个东西，就要有字符串
    • 设置日期 date -s “日期”

  • cal 显示当前日历

  • 搜索查找类

    • find【搜索范围】【选项】 从指定目录向下递归地遍历各个子目
      • -name 文件名 按照指定的文件名查找文件，如果文件名不固定，查找某一类型 用 *.txt
      • -user 用户名 查找属于指定用户名所有文件
      • -size 文件大小 按照指定的文件大小查找文件 “+”代表大于，“-”代表小于
    • locate指令
      • locate可以快速定位文件路径，利用实现建立的系统中所有文件名称及路径的locate数据库快速定位给定的文件，无需遍历整个文件系统，查询速度较快。为了保证查询结果的准确度，管理员必须定期更新locate时刻
      • 第一次运行前，必须用updatedb指令创建locate数据库
    • which 查看某个指定在哪个目录下
    • grep指令和管道符号 | 管道符可以连续使用 grep【选项】查找内容 源文件
      • grep过滤查找，管道符“|” 表示将前一个命令的处理结果
      • 常用选项
        • -n显示匹配行及行号
        • -i 忽略字母大小写
      • 例：在hello.txt 中查找“yes”所在行，并显示行号
        • cat /home/hello.txt | grep -n “yes”
        • grep -n “yes” /home/hello.txt

    压缩和解压类

    • gzip / gunzip 压缩和解压 这个只是对文件进行的操作 这个是文件操作之后原文件就没了
      • gzip 文件 压缩文件，只能压缩为*.gz文件
      • gunzip 文件.gz 解压缩文件命令
    • zip/ unzip 压缩和解压 这个操作之后原文件还在
      • zip【选项】XXX.zip 将要压缩的内容（功能描述：压缩文件和目录的命令）
        • -r 递归压缩，即压缩目录
      • unzip 【选项】 XXX.zip （功能描述：解压缩文件）
        • -d 目录 ：指定解压后文件的存放目录
    • tar 压缩和解压都是这个指令 最后打包后的文件是.tar.gz的文件 可压文件，也可压目录
      • tar【选项】 XXX.tar.gz 打包的内容 如何确定是压缩还是解压是通过选项来实现的
        • -c 产生.tar打包文件
        • -v 显示详细信息
        • -f 指定压缩后的文件名
        • -z 打包同时压缩
        • -x 解包.tar文件
      • 例：压缩多个文件，将/home/pig.txt 和 /home/cat.txt 压缩成 pc.tar.gz
        • tar -zcvf pc.tar.gz /home/pig.txt /home/cat.txt 如果是解压，指令推荐zxvf
      • 例：myhome.tar.gz 解压到 /opt/tmp2下
        • mkdir/opt/tmp2
        • tar -zxvf /home/myhome.tar.gz -C /opt/tmp2
