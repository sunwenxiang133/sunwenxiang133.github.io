基础了解

给的api怎么读

behavior是设置这个组件有什么样子的功能

model是数据绑定可以绑什么东西，绑定之后有什么功能

style是样式

css文件夹

app里面是全局的css

quasar.variables里面是定义的全局的颜色变量（quasar自带的组件用的就是这个主题变量）

quasar.config.js

css

全局搜css,定位到，然后把想要全局引入的东西放到样式表里面即可（但是想要引入的东西要是在css文件夹下面的东西）

boot

boot文件夹里面是需要用的组件，在boot文件夹里面写完组件，要在quasar.config.js里面的boot写进去

extras

extras里面配置的是需要用的图标

有个默认没有的但是很常用的

bulid 和编译相关的

路由的模式默认是哈希

可以通过这里更改webpack的配置

devServer

开发时开的服务器，代理服务器的设置和配置

framework

// lang: 'en-US', // Quasar language pack 配置语言的，后面组件如果用到这个会使用不同的语言

中文是 zh-CN

plugins 配置需要使用的quasar的插件，添加完插件还要在config配置项里面写上对应的配置

Quasar插件是您可以在Vue文件中以及在其外部使用的功能，如Notify、BottomSheet、AppVisibility等。

要使用它们，您需要在/quasar.conf.js中添加对它们的引用：

animations

动画

如果用animations: 'all'是引入所有的动画

如果只要用部分的动画，就在animation里面的配置上面写上即可

使用方式

boot配置 (启动文件)

因为quasar没有main.js（为了多端适配，所以去掉了，用boot来代替）

Quasar应用程序的一个常见用例是在实例化根Vue应用程序实例之前运行代码，例如注入和初始化您自己的依赖项（例如：Vue组件、库…）或者只是配置一些启动代码应用程序。

由于您无法访问任何/main.js文件（因此Quasar CLI可以无缝初始化并为SPA/PWA/SSR/Cordova/Electron构建相同的代码库）Quasar提供了该问题的优雅解决方案——允许用户定义所谓的启动文件。

文件可以用quasar来生成一个新启动文件

最后一步是告诉Quasar使用你的新启动文件。 为了做到这一点，你需要在/quasar.conf.js中添加启动文件

引入boot启动文件的例子

import引入需要引入的东西

然后用app.use(xxx)使用即可

Flex Grid

Flexbox

直接在css的父类上面加上这些类就是对应的布局方式

不同的类对应不同组件方式

Grid Row

栅格系统

默认是12份

这个也支持断点

offset前面空多少份

Grid Gutter

加上这个之后，它的所有子元素会有一个默认的边距

q-gutter-md 加上外边距

q-col-gutter-md加上内边距

Flex Playground

这是一个方便的可视化的界面

Layout

要头部

要尾部

要左弹出框

要右弹出框

要头部下面的切换页面的标签

h,f 大写，小写代表绝对定位还是相对定位

四个角的选项代表分别布局在哪些位置

Header Reval标题的头上滑一定位置会消失

Behavior还可以选择模式，是在手机模式（侧边框会占很多的位置）还是在其他的模式

Left-side 中的Overlay mode是侧边框会不会把其他的位置挤出去

Separator type是控制没有没阴影

设置完成的东西直接导出，然后赋值粘贴到

q-page

这个是布局用的，就包在最外层就行

QPageSticky组件有助于将由它包裹的DOM元素/组件放置到QPage内容区域中的静态位置，无论用户在哪里滚动。
